# Model Approval Policy

## Purpose

This policy describes the approval expectations for ML and GenAI systems before deployment.

## Approval Criteria

Future versions will define approval criteria for risk tier, data sensitivity, model purpose, validation results, access controls, monitoring coverage, and cost thresholds.

## Required Evidence

- Model or AI system record
- Owner and reviewer details
- Risk tier
- Validation summary
- Monitoring plan
- Model card or equivalent documentation

## Future Expansion

- Approval workflow states
- Required approver roles
- Production release checklist
- Reapproval triggers
