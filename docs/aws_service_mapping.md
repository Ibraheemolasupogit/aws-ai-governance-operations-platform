# AWS Service Mapping

## Purpose

This document explains how planned local platform capabilities map to AWS services conceptually.

## Planned Mapping

- IAM and AWS Organizations: identity, ownership, and access review context
- CloudTrail: audit event concepts
- CloudWatch: health, metrics, logs, and alert concepts
- AWS Config: configuration and compliance concepts
- S3: evidence and report storage concepts
- SageMaker Model Registry: model catalogue concepts
- Bedrock: GenAI workload governance concepts
- Budgets and Cost Explorer: cost monitoring concepts
- EventBridge, Lambda, and Step Functions: workflow automation concepts
- DynamoDB: metadata storage concepts
- QuickSight: executive reporting concepts

## Current Constraint

This repository does not connect to AWS or create cloud resources in Milestone 1.
